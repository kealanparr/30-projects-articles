7. Learn more TypeScript

Through my career I have worked with a wide range of different programming languages, framework, libraries or patterns - and I've always enjoyed learning new things

For one role, I spent 2 years doing pure TypeScript development, and consider myself a talented TypeScript developer

There is always more to learn though, and there were some features of the language I was less familiar with

I decided to try and learn some of the less commonly used, and difficult features of TypeScript

Advanced features like:

  • Infer keyword
  • Partials
  • Type algebra
  • Conditional types
  • Distributive conditional types
  • Index signatures
  • Function overloads
  • Hybrid Types
  • Assertion functions
  • Mapped Types
  • Type Predicates
  • Generic constraints
  • keyof and typeof
  • Recursive types

The whole point of 30-projects is to code more, build more and ship more - so following along a course wasn't quite in the spirit of the challenge

I wanted to be "hands-on" and either build something with these features, or complete coding exercises by using the features

This was when I discovered Execute Program (I'm not affiliated, or paid in any way)

I could get hands on "practice" learning the features, and the curriculum would only let you move on if you complete exercises (and pass the tests)

This seemed much closer to what I wanted, so for the past few months I have been working through their TypeScript courses

Whilst I initially started with the TypeScript challenges I ended up doing more courses than I expected!
